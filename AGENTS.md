# AGENTS.md - 開発エージェント向け指示書

このプロジェクトでは、開発環境の一貫性を保つため、`golang`, `wails`, `node.js` などのツールチェーンのバージョン管理を **`mise`** に統一します。

## ルール

1.  **ツールチェーンの導入**:
    *   `golang`, `wails`, `node.js` などの必要なツールは、すべて `mise.toml` ファイルで定義・管理します。
    *   開発を開始する前、またはツールが不足している場合は、必ず `mise install` コマンドを実行して、定義されたバージョンのツールをインストールしてください。

2.  **コマンドの実行**:
    *   `go`, `npm`, `wails` などのコマンドは、原則として直接実行せず、`mise x -- <command>` のように `mise` を経由して実行します。
    *   これにより、プロジェクトで定義された正しいバージョンのツールが常に使用されることを保証します。
    *   例:
        *   `mise x -- go version`
        *   `mise x -- npm install`
        *   `mise x -- wails dev`

このルールに従うことで、どの開発環境でも同じバージョンのツールセットでビルドと実行が行われ、環境差異による問題を未然に防ぎます。
